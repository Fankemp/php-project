# 📦 Умный парсер данных API (Laravel)

Сервис для автоматического сбора, синхронизации и хранения данных о заказах, продажах, поставках и складских остатках через API.

Проект спроектирован с упором на отказоустойчивость, работу с большими объемами данных и строгую консистентность данных.

---

## 🚀 Ключевые архитектурные решения

### Идемпотентность парсинга (`updateOrCreate`)

Скрипт можно запускать любое количество раз. Он не создает дубликатов записей, а обновляет существующие данные при изменении статуса.

Пример:

* Заказ был создан.
* Позже клиент отменил заказ.
* При повторном запуске парсера запись будет обновлена, а не продублирована.

### Оптимизация работы с БД (Транзакции и чанки)

Данные разбиваются на пакеты (`chunks`) и сохраняются внутри `DB::transaction()`.

Преимущества:

* снижение нагрузки на базу данных;
* увеличение скорости массовой записи;
* защита от частичного сохранения данных при ошибках.

### Обход лимитов API (Rate Limiting)

Между запросами к API используются микро-паузы (`usleep`), предотвращающие получение ошибки:

```http
429 Too Many Requests
```

### Промышленная структура базы данных

* Идентификаторы заказов и поставок хранятся как `string` или `bigInteger`.
* Поддерживаются длинные Snowflake ID и другие нестандартные идентификаторы.
* Потенциально отсутствующие поля API обрабатываются через `nullable`.
* Реализована защита от неконсистентных данных внешнего сервиса.

### UX консольного приложения

Используются интерактивные прогресс-бары Symfony Console для отображения процесса загрузки данных в реальном времени.

## 🗄 Структура базы данных

Система состоит из четырех основных таблиц:

### `incomes`

Поставки товара на склад.

### `stocks`

Текущие складские остатки.

### `orders`

Заказы покупателей.

### `sales`

Продажи, выкупы и возвраты.

### Связь данных

Товары связываются между таблицами через:

* `nm_id` — артикул товара;
* `barcode` — штрихкод товара.

## ⚙️ Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/Fankemp/php-project.git
cd php-project
```

### 2. Установка зависимостей

```bash
composer install
```

### 3. Настройка окружения

Создайте файл `.env`:

```bash
cp .env.example .env
```

Сгенерируйте ключ приложения:

```bash
php artisan key:generate
```

Укажите параметры подключения к базе данных в файле `.env`.

### 4. Запуск миграций

```bash
php artisan migrate
```

## 💻 Использование

Основная команда запуска парсера:

```bash
php artisan api:parse
```

### Доступные параметры

| Параметр                | Описание                      |
| ----------------------- | ----------------------------- |
| `--dateFrom=YYYY-MM-DD` | Дата начала парсинга          |
| `--dateTo=YYYY-MM-DD`   | Дата окончания парсинга       |
| `--only=table_name`     | Парсинг только одной сущности |

### Поддерживаемые значения `--only`

```text
orders
sales
incomes
stocks
```

---

## 📌 Примеры использования

### Скачать все данные за последний месяц

```bash
php artisan api:parse --dateFrom=2024-05-01
```

### Скачать только складские остатки

```bash
php artisan api:parse --only=stocks
```

### Скачать только заказы за конкретный день

```bash
php artisan api:parse \
    --only=orders \
    --dateFrom=2024-06-01 \
    --dateTo=2024-06-01
```
